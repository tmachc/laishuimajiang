## 修改目标
1. 将胡牌逻辑改为"坎胡"：只能胡顺子中间的那张牌
2. 去掉血流成河、血战到底模式，只保留普通模式

## 坎胡核心规则
胡牌的那张牌必须**只能**作为顺子中间那张牌，不能有其他任何胡法。

### 需要排除的其他胡法类型：
| 类型 | 条件 | 示例 |
|------|------|------|
| 将牌 | 手上有同牌>=1张 | 2334胡3 → 33将牌 ❌ |
| 顺子边张(左) | 手上有A+1,A+2 | 23胡1 → 123 ❌ |
| 顺子边张(右) | 手上有A-1,A-2 | 45胡6 → 456 ❌ |
| 刻子 | 手上有同牌>=2张 | 2224胡2 → 222刻子 ❌ |

## 坎胡算法伪代码

```javascript
function checkKanHu(seatData, targetPai) {
    // 1. 检查是否可以作为顺子中间牌
    if (!canBeMiddleOfShunzi(seatData, targetPai)) {
        return false;  // 不能组成顺子
    }
    
    // 2. 检查是否有其他胡法
    // 2.1 检查将牌
    if (hasSameCard(seatData, targetPai, 1)) {
        return false;  // 可以作为将牌
    }
    
    // 2.2 检查顺子边张(左) - A+1, A+2
    if (hasCards(seatData, [targetPai+1, targetPai+2])) {
        return false;  // 可以作为顺子第一张
    }
    
    // 2.3 检查顺子边张(右) - A-1, A-2
    if (hasCards(seatData, [targetPai-1, targetPai-2])) {
        return false;  // 可以作为顺子第三张
    }
    
    // 2.4 检查刻子
    if (hasSameCard(seatData, targetPai, 2)) {
        return false;  // 可以作为刻子
    }
    
    return true;  // 只能坎胡
}
```
